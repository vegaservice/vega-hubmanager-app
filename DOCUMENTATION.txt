WHAT THIS APP DOES — COMPLETE FUNCTIONALITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

──────────────────────────────────
1. AUTHENTICATION
──────────────────────────────────

✅ DONE — Phone OTP via Firebase Auth

Only workers with role = "hub_manager" can login.

On OTP verification:
  → Checks workers collection where phone == entered_number
  → AND where role == "hub_manager"
  → If not found: "Not registered as Hub Manager"
  → If found: Login success

FCM token registered for push notifications.

──────────────────────────────────
2. DASHBOARD TAB 🏠
──────────────────────────────────

✅ DONE — Real-time live dashboard

STATS SHOWN:
  📦 Today's Bookings (count)
  💰 Today's Revenue (₹)
  ⚡ In Progress jobs (count)
  🚨 Unassigned bookings (count — red alert if > 0)
  ✅ Available staff (count)
  ⭐ Average rating (from completed jobs)

URGENT ALERTS (top of screen, red banner):
  → If unassigned bookings exist: "X Bookings Need Assignment!"
  → Tapping takes directly to Bookings tab, filtered to Unassigned
  → If open complaints: "X Open Complaints" (gold banner)

LIVE WORKER STATUS:
  → Shows all team members
  → Green dot = Free/Available
  → Orange dot = On Job (isAvailable = false)
  → Red = Suspended/Blocked
  → One-tap call button for each worker

RECENT BOOKINGS:
  → Last 5 bookings with status badges
  → Shows customer name, order ID, revenue, time
  → Tap any to open full booking detail

──────────────────────────────────
3. BOOKINGS TAB 📦
──────────────────────────────────

✅ DONE — Complete booking management

FILTER TABS:
  All         → All bookings ever (real-time)
  Unassigned  → Status = "confirmed" — needs worker
  Active      → Status = assigned / on_the_way / in_progress
  Completed   → Status = "completed"

EACH BOOKING CARD SHOWS:
  - Order ID
  - Customer name + phone
  - Address
  - Service type
  - Scheduled time/slot
  - Status badge (colored)
  - Assigned worker name (or "Needs assignment" in red)
  - Revenue amount (₹) — Hub manager CAN see this
  - Delay flag warning ⚠️
  - Time since booking

QUICK ASSIGN BUTTON:
  → If status = "confirmed" → "Assign Now" button on card
  → Opens assignment screen directly

──────────────────────────────────
4. JOB DETAIL SCREEN
──────────────────────────────────

✅ DONE — Full booking detail

INFORMATION VISIBLE:
  Customer name, phone, address, slot
  Service items + prices (hub manager can see)
  Total amount ₹
  OTP (4-digit code to start service)
  Status timeline
  Photo proof count (Before: X, After: X)
  Assigned worker info
  Delay flags

ACTIONS:
  📞 Call Customer (one tap → phone dialer)
  📞 Call Worker (one tap → phone dialer)
  👩 Assign Professional → opens smart assignment
  🔄 Reassign (if already assigned)
  ⚡ Update Status (manual override — all statuses)

──────────────────────────────────
5. SMART WORKER ASSIGNMENT
──────────────────────────────────

✅ DONE — Intelligent worker suggestion

When assigning a worker, the system:
  1. Filters: isAvailable = true AND status = "active"
  2. Sorts by: ratingAvg DESC, then totalJobsCompleted DESC
  3. Displays sorted list with best match first
  4. Shows "⭐ BEST MATCH" gold badge on top worker

Worker card shows:
  - Name and first letter avatar
  - Area and star rating
  - Jobs done count
  - Free/Busy status badge

On selecting a worker:
  → Firestore update:
      jobs/{jobId}.assignedWorkerId = worker.id
      jobs/{jobId}.assignedWorkerName = worker.name
      jobs/{jobId}.assignedWorkerPhone = worker.phone
      jobs/{jobId}.status = "assigned"
      jobs/{jobId}.assignedAt = server timestamp

REASSIGN:
  → Same flow — can reassign at any time
  → New worker gets the job
  → Previous assignment overwritten

──────────────────────────────────
6. TEAM MANAGEMENT TAB 👥
──────────────────────────────────

✅ DONE — Full team visibility and control

TEAM LIST:
  All workers in the hub
  Shows: Name, phone, area, rating, jobs done
  Status dots: Active/Free, Active/On Job, Suspended, Blocked
  One-tap call button
  Tap any worker → Worker Detail screen

ADD WORKER:
  → "+" button opens Add Worker form
  → Fields: Name, Phone, Role (Worker/Hub Manager), Area
  → Creates worker document in Firestore
  → Worker can immediately login with their phone

WORKER DETAIL SCREEN:
  - Full profile (name, phone, status, rating)
  - Performance stats (jobs done, performance score)
  - Attendance data
  - Earnings visible to hub manager
  - Worker control panel (Active/Suspend/Block)

WORKER CONTROL (3 states):
  ● ACTIVE   → Normal state, can login, receives jobs
  ● SUSPENDED → Cannot receive new jobs, can still login
  ● BLOCKED  → Cannot login at all (enforced in Worker App)

When you block a worker:
  → workers/{id}.status = "blocked"
  → If blocked worker tries to login → "Account Blocked" error
  → They cannot access the app at all

──────────────────────────────────
7. COMPLAINTS TAB 🧾
──────────────────────────────────

✅ DONE — Complete complaint management

COMPLAINTS LIST:
  Shows all complaints from complaints collection
  Status badges: Open (red), In-Progress (gold), Resolved (green)
  Customer name, job ID, issue text, time

COMPLAINT DETAIL:
  Full complaint information
  Resolution text input
  "✅ Mark Resolved" button

ON RESOLVING:
  → complaints/{id}.status = "resolved"
  → complaints/{id}.resolution = text entered
  → complaints/{id}.resolvedAt = timestamp

Open complaints count shown as badge on tab icon.

NOTE: Complaints are created by customers or admin.
Hub Manager can view and resolve them.

──────────────────────────────────
8. REVENUE VISIBILITY
──────────────────────────────────

✅ DONE — Hub Manager CAN see all pricing

Unlike Worker App, Hub Manager sees:
  - Job amounts (jobs/{jobId}.amount or .total)
  - Daily revenue on dashboard
  - Worker earnings (in Worker Detail screen)

This data is NEVER sent to Worker App.

──────────────────────────────────
9. PUSH NOTIFICATIONS
──────────────────────────────────

✅ DONE — FCM receiving
  - Token saved on login
  - Foreground alerts shown

❌ NOT DONE — FCM sending (needs Cloud Functions)
  Same limitation as Worker App.
  Hub Manager needs notifications for:
    → New customer booking arrives
    → Worker rejected a job
    → Delay flag triggered

10. PROFILE TAB
──────────────────────────────────

✅ DONE:

  - Hub statistics (total jobs, revenue, team size)

  - Logout

FIRESTORE COLLECTIONS USED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

READS FROM:
  workers        → All workers in hub (real-time)
  jobs           → All bookings (real-time)
  complaints     → All complaints (real-time)

WRITES TO:
  jobs/{jobId}:
    assignedWorkerId, assignedWorkerName, assignedWorkerPhone
    status, assignedAt, reassignedAt
    [any status field via manual override]

  workers/{id}:
    status (active/suspended/blocked)
    isAvailable
    name, phone, role, area, services
    salary, performanceScore

  complaints/{id}:
    status, resolution, resolvedAt

SCALABILITY — READY FOR 500+ CUSTOMERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Firestore real-time listeners:
  → Jobs listener: loads last 100 bookings
  → Workers listener: loads all active workers
  → Complaints listener: loads last 50

For high-volume (500+ bookings/day):
  → Increase .limit(100) to .limit(500) in the listeners
  → Add index on (createdAt, status) in Firebase Console
  → Firestore auto-scales — no server changes needed

Firebase Blaze plan supports:
  → Unlimited reads/writes
  → Automatic horizontal scaling
  → No downtime during traffic spikes
